# MENU_SETUP

## Status
Menu Setup v1

## Purpose
This file defines the menu structure for the New Earth website in WordPress.

The goal is to keep navigation:
- clear
- calm
- scalable
- aligned with the Blueprint

---

## CORE RULE

The main navigation should remain simple.

Do not overload the top menu with every subpage.
Use the top menu to establish orientation.
Use child pages, internal links, and footer menus for deeper structure.

---

## FUTURE SUBMENU / CHILD PAGE LOGIC

For version 1, keep child pages mostly out of the top menu.

Instead, use internal links from parent pages.

### Blueprint child pages
These can appear later inside the Blueprint page itself or as dropdowns if needed:
- The 12 Pillars
- Inner Blueprint
- Transition Path
- New Earth Test

### Pillar detail pages
Do not add all 12 pillar pages to the main menu.
Link to them from:
- Blueprint
- Systems
- Journal
- future resources

### MicroGrow child pages
These can appear later:
- Features
- How It Works
- Roadmap
- Blueprint Alignment
- FAQ

Again, keep these off the top menu unless the product section grows significantly.

### Step 1

### Step 3
Create the main menu using this order:
- Home
- Manifesto
- Blueprint
- Systems
- MicroGrow
- Journal
- About
- Contact

### Step 4

- Manifesto

Assign the correct display locations depending on theme support.

---

## NAVIGATION PRINCIPLES

### 1. Calm over clutter
The navigation should not feel busy.

### 2. Doctrine before detail
Blueprint and Systems deserve top-level visibility.

### 3. Product within philosophy
MicroGrow should be visible, but within the wider New Earth structure.

### 4. Depth through pages, not menu overload
Let pages carry depth.
Do not force all depth into navigation.

---

## FUTURE EXPANSION

If the site grows later, possible additions include:
- Resources
- Projects
- Guides
- Downloads

These should only be added when they exist as real, useful sections.

## FINAL MENU TEST

Before publishing, confirm:

1. Is the main menu simple and understandable?
2. Does the menu reflect the true architecture of the site?
3. Is Blueprint given proper importance?
4. Does MicroGrow feel integrated rather than random?
5. Are legal pages available in the footer?
6. Can a new visitor understand where to go next?
