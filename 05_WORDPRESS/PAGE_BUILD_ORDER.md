# PAGE_BUILD_ORDER

## Status
WordPress Build Order v1

## Purpose
This document defines the order in which pages should be created and built in WordPress.

The goal is to prevent random page creation and ensure that the public website is constructed in a stable, expandable sequence.

---

## Build principle

Pages should be built in the order that best supports:

- clarity
- structure
- doctrine alignment
- smooth internal linking
- future expansion

Do not build pages based only on impulse or appearance.

Build the foundation first.

---

## Phase 1 — WordPress foundation

Before creating public pages, complete the basic WordPress setup.

### Tasks
- install WordPress on IONOS
- confirm domain connection
- confirm SSL
- set site title and tagline
- set permalink structure
- remove default sample content
- create essential admin accounts
- install the chosen parent theme
- prepare the New Earth child theme
- install only essential plugins
- configure media defaults
- configure basic site settings

### Outcome
A clean WordPress environment ready for structured page creation.

---

## Phase 2 — Create top-level shell pages

Create the main top-level pages first, even if content is not fully pasted yet.

### Create these pages
- Home
- Manifesto
- Blueprint
- Systems
- MicroGrow
- Journal
- About
- Contact
- Privacy Policy
- Terms
- Cookie Policy
- Disclaimer

### Outcome
The main site skeleton exists inside WordPress.

---

## Phase 3 — Set navigation

Build the menu structure before styling every page in detail.

### Main menu
- Home
- Manifesto
- Blueprint
- Systems
- MicroGrow
- Journal
- About
- Contact

### Footer menu
- Manifesto
- Blueprint
- Systems
- MicroGrow
- Journal
- About
- Contact
- Privacy Policy
- Terms
- Cookie Policy
- Disclaimer

### Outcome
The site has a usable navigation structure early in the process.

---

## Phase 4 — Build the Home page

The Home page should be built first because it defines tone, direction, and user flow.

### Home page goals
- establish New Earth clearly
- introduce the spirit of the project
- create trust
- present the key pathways
- bridge visitors to Manifesto, Blueprint, Systems, and MicroGrow

### Home page section order
1. Hero
2. Introduction to New Earth
3. Why a new system is needed
4. Core principles or summary values
5. Blueprint introduction
6. Systems overview
7. MicroGrow feature section
8. Journal or latest insights
9. Closing call to action

### Outcome
The site has a meaningful entry point.

---

## Phase 5 — Build the Manifesto page

The Manifesto page should come next because it defines the spirit and worldview behind the site.

### Manifesto page goals
- communicate the moral and philosophical basis of New Earth
- create emotional and spiritual resonance
- frame the Blueprint as the next logical step

### Manifesto page section order
1. Page introduction
2. Foundational worldview
3. Core values
4. Human and Earth relationship
5. Why old systems fail
6. Invitation into New Earth
7. Call to continue into Blueprint

### Outcome
The visitor can feel the vision before being introduced to structure.

---

## Phase 6 — Build the Blueprint page

This is one of the most important pages in the whole site.

### Blueprint page goals
- present the New Earth Blueprint clearly
- introduce the 12 pillars
- show that this is a structured framework, not vague philosophy
- connect inner development with outer systems

### Blueprint page section order
1. Hero / title
2. Foundational vision
3. Core principles
4. Purpose of civilization
5. Overview of the 12 pillars
6. Inner Blueprint
7. Transition Path
8. New Earth Test
9. Closing declaration

### Outcome
The website’s doctrine is visible and structured.

---

## Phase 7 — Build the Systems page

Once the Blueprint exists, build the Systems page.

### Systems page goals
- explain how the Blueprint translates into functional systems
- provide a bridge between philosophy and implementation
- introduce categories such as governance, knowledge, product, and community systems

### Systems page section order
1. Systems overview
2. Why systems matter
3. New Earth systems categories
4. Relationship to the 12 pillars
5. Current implementation direction
6. Future system map

### Outcome
Visitors can understand that New Earth is a design framework, not only an idea.

---

## Phase 8 — Build the MicroGrow page

The MicroGrow page should be built after the philosophical and systems layers are clear.

### MicroGrow page goals
- position MicroGrow as the first practical implementation node
- connect the product to the wider New Earth system
- show why it matters
- explain current functionality and future direction

### MicroGrow page section order
1. Hero
2. What MicroGrow is
3. Why it exists
4. Problem it addresses
5. Key features
6. Blueprint alignment
7. Roadmap
8. FAQ
9. Call to action

### Outcome
MicroGrow feels grounded within the wider architecture.

---

## Phase 9 — Build the About page

The About page should build trust after the main system pages are in place.

### About page goals
- explain the origin and purpose of New Earth
- add human context
- clarify the deeper intent behind the project

### About page section order
1. About New Earth
2. Why it exists
3. Founding vision
4. Relationship between spiritual intelligence and systems design
5. Future direction
6. Contact bridge

### Outcome
Visitors can understand the project on a more personal level.

---

## Phase 10 — Build the Contact page

The Contact page should be simple, clear, and intentional.

### Contact page goals
- provide a direct route for aligned communication
- invite collaboration

### Contact page section order

2. Contact invitation
3. Inquiry types

5. Closing note

### Outcome
The site becomes actionable.

## Rule for all page builds

Before building any page in WordPress, confirm:

1. the page exists in the sitemap
2. the page has a content source file
3. the page has a defined role
4. the page links naturally to other core pages
5. the page supports the long-term New Earth architecture
