# PLUGIN_STACK

## Status
Plugin Stack v1

## Purpose
This file defines the recommended plugin approach for the New Earth website.

The goal is to keep WordPress:
- clean
- stable
- secure
- maintainable
- lightweight

---

## CORE RULE

Use as few plugins as possible.

Every plugin should have a clear purpose.
If a plugin does not solve a real problem, do not install it.

New Earth should be built on:
- strong structure
- clear content
- careful theme setup
- minimal plugin dependency

Not on plugin overload.

---

## RECOMMENDED PLUGIN CATEGORIES

### 1. SEO
Use one SEO plugin only.

Purpose:
- meta titles
- meta descriptions
- sitemap support
- indexing control
- schema basics

Rule:
Do not install multiple SEO plugins.

---

### 2. Forms
Use one form plugin only.

Purpose:
- contact form
- inquiry form
- future collaboration forms

Rule:
Keep forms simple and focused.

---

### 3. Security
Use one lightweight security solution.

Purpose:
- login protection
- brute force reduction
- admin hardening
- basic monitoring

Rule:
Avoid security plugins that overwhelm the site with noise.

---

### 4. Backup
Use one backup solution.

Purpose:
- restore capability
- peace of mind
- pre-update backup points
- migration safety if needed

Rule:
Backups matter more than plugin quantity.

---

### 5. Performance / Caching
Only add this if needed.

Purpose:
- improve page speed
- reduce unnecessary load
- optimize delivery

Rule:
If IONOS already covers enough performance for v1, do not overcomplicate it.

---

## PLUGINS TO AVOID EARLY

Avoid installing these too early unless clearly needed:

- heavy visual builders
- multiple design plugins doing the same job
- excessive animation plugins
- complex popup systems
- social clutter plugins
- unnecessary analytics add-ons
- duplicate SEO tools
- bloated “all-in-one” toolkits

---

## MINIMUM SAFE STACK FOR V1

For version 1, the website only truly needs:

- 1 SEO plugin
- 1 forms plugin
- 1 backup plugin
- 1 security plugin
- optional caching plugin if required

That is enough.

---

## PLUGIN DECISION RULE

Before installing any plugin, ask:

1. What exact problem does this solve?
2. Is there already another tool solving this?
3. Can this be handled by the theme or WordPress itself?
4. Does this add complexity I do not need yet?
5. Will this still make sense six months from now?

If the answer is unclear, do not install it yet.

---

## NEW EARTH PLUGIN PHILOSOPHY

The plugin stack should reflect New Earth principles:

- clarity over clutter
- function over noise
- stewardship over excess
- maintainability over convenience
- resilience over dependency

---

## V1 PLUGIN CHECKLIST

Before launch, confirm:

- only essential plugins are active
- unused plugins are deleted
- plugin overlap does not exist
- every plugin has a written reason for being installed
- backups work
- contact form works
- SEO basics are configured
- security basics are active

---

## FINAL RULE

A lean WordPress stack is a strong WordPress stack.
