# MEDIA_LIBRARY_RULES

## Status
Media Library Rules v1

## Purpose
This file defines how media should be organized and used for the New Earth website.

The goal is to keep media handling:
- clean
- searchable
- consistent
- scalable
- visually coherent

A messy media library quickly creates friction.
This file prevents that.

---

## CORE RULE

Do not upload media randomly.

Every media item should have:
- a purpose
- a clear filename
- a place in the visual system
- a connection to a page, section, or future reuse case

---

## MEDIA CATEGORIES

Organize media conceptually using these groups:

### 1. Brand assets
Examples:
- primary logo
- alternate logo
- icon mark
- favicon
- social avatar

### 2. Core page imagery
Examples:
- Home hero imagery
- Manifesto supporting imagery
- Blueprint support graphics
- Systems diagrams

### 3. MicroGrow imagery
Examples:
- product renders
- device photos
- diagrams
- interface mockups
- feature visuals

### 4. Journal imagery
Examples:
- article thumbnails
- category visuals
- diagrams for posts

### 5. Legal / utility assets
Examples:
- icons
- small support graphics
- downloadable PDF covers later

---

## FILE NAMING RULES

Use descriptive lowercase filenames with hyphens.

### Good examples
- new-earth-home-hero-forest-light.jpg
- new-earth-blueprint-pillars-diagram.png
- microgrow-controller-front-view.jpg
- microgrow-system-map-v1.png

### Avoid
- IMG_0384.jpg
- screenshot-final-new-new.png
- hero1.png
- randomfile.webp

---

## ALT TEXT RULE

Every important image should have useful alt text.

Alt text should describe:
- what the image is
- what purpose it serves

Examples:
- Forest sunrise used as a calming hero image for the New Earth homepage
- Diagram showing the 12 pillars of the New Earth Blueprint
- MicroGrow prototype controller displayed on a desk

Do not leave important images without alt text.

---

## IMAGE STYLE RULES

The media used on the site should feel visually aligned with the New Earth identity.

Preferred imagery qualities:
- natural
- calm
- warm
- spacious
- grounded
- reflective
- intelligent

Avoid imagery that feels:
- generic stock-corporate
- overly glossy
- hyper-techno
- chaotic
- spiritually cliché
- visually cluttered

---

## BRAND ASSET RULES

Brand assets should be uploaded in a controlled way.

Core brand assets to prepare:
- main logo
- light version if needed
- dark version if needed
- favicon
- simple icon mark

These should use clear filenames and remain easy to locate.

Suggested naming:
- new-earth-logo-primary.svg
- new-earth-logo-dark.svg
- new-earth-favicon.png

---

## MICROGROW MEDIA RULES

MicroGrow visuals should feel like part of New Earth, not like a disconnected product brand.

That means:
- use the same visual tone
- keep backgrounds simple
- avoid generic “tech startup” product styling
- favor clarity and grounded realism
- support stewardship and systems thinking

Suggested media types:
- clean prototype photos
- system diagrams
- sensor/control concept diagrams
- future interface mockups

---

## IMAGE OPTIMIZATION RULES

Before upload:

- resize overly large images
- compress where appropriate
- use web-friendly formats
- avoid uploading huge raw files directly

The goal is:
- faster loading
- cleaner media library
- easier reuse

---

## DIAGRAM RULES

Diagrams are important for New Earth.

Likely future diagrams include:
- 12 pillars overview
- New Earth systems map
- Blueprint -> Systems -> MicroGrow pathway
- transition path timeline
- MicroGrow architecture diagrams

Rules for diagrams:
- keep them readable
- use the New Earth visual tone
- use clean filenames
- keep versions labeled clearly

Example:
- new-earth-12-pillars-diagram-v1.png
- microgrow-architecture-map-v1.png

---

## WORDPRESS MEDIA HABITS

When uploading:

1. upload intentionally
2. rename files before upload
3. add alt text
4. place media on the right page
5. do not upload duplicates unless needed

If a replacement image is required, update carefully rather than creating unnecessary clutter.

---

## SUGGESTED FIRST MEDIA SET

Before or during first build, gather:

### Brand
- logo
- favicon

### Home
- one calm hero image
- one supporting nature image if needed

### Blueprint
- one future pillars diagram placeholder

### Systems
- one systems-map placeholder

### MicroGrow
- one product image or diagram placeholder

This is enough to build version 1 without overcomplicating the media library.

---

## FUTURE DOWNLOAD ASSETS

Later media may also support:
- PDFs
- downloadable guides
- printable diagrams
- social share graphics
- presentation covers

When that happens, create a naming pattern and keep it consistent.

---

## FINAL MEDIA TEST

Before uploading any file, ask:

1. What page or section is this for?
2. Is the filename clear?
3. Does this image fit the New Earth visual tone?
4. Is this actually needed?
5. Will I still be able to find this easily later?

If not, fix it before upload.

---

## FINAL RULE

A clean media library is part of a clean system.
