# PAGE_LAYOUT_RULES

## Status
Page Layout Rules v1

## Purpose
This file defines the layout rules for the New Earth website.

The goal is to create page structures that feel:
- spacious
- coherent
- calm
- readable
- modular
- scalable

These rules should guide all major pages so the website feels like one system rather than a collection of disconnected pages.

---

## CORE LAYOUT PRINCIPLE

Each page should feel like a guided pathway, not a pile of sections.

Visitors should feel oriented as they move down the page.

A New Earth page should create:
- clarity
- rhythm
- progression
- breathing room
- meaningful transitions

---

## PAGE FLOW MODEL

Most pages should follow this general flow:

1. Hero
2. Orientation / introduction
3. Core explanation
4. Deeper structure
5. Practical bridge
6. Closing declaration or CTA

This creates a calm reading rhythm and makes pages easier to build consistently.

---

## SECTION RHYTHM RULE

Every section should have a clear reason to exist.

A section should do one of the following:
- introduce
- explain
- deepen
- connect
- transition
- invite action

If a section does not serve one of these roles, simplify or remove it.

---

## PAGE WIDTH RULES

### Content width
Keep standard readable content relatively narrow.

Reason:
Long line lengths reduce readability and make reflective content feel heavy.

### Wide sections
Use wider layouts only when needed for:
- pillar card grids
- feature grids
- multi-column sections
- timeline blocks

### Visual principle
The site should feel open and breathable, not stretched and sparse.

---

## SECTION PADDING RULES

### Top and bottom spacing
Use generous vertical spacing between major sections.

Reason:
New Earth content needs room to breathe.
The site should not feel cramped or rushed.

### Internal spacing
Within each section, headings, paragraphs, lists, and CTAs should have consistent spacing.

### Rule
Spacing should create calm structure, not empty drift.

---

## HERO SECTION RULES

Every major page should begin with a clear hero.

### Hero must include
- eyebrow or context line
- main heading
- supporting text
- one or two clear actions

### Hero should feel
- focused
- spacious
- stable
- emotionally clear

### Hero should not feel
- overloaded
- crowded with too many buttons
- visually noisy
- dependent on gimmicks

---

## TEXT SECTION RULES

Text-heavy sections are central to New Earth.

### Use:
- one clear heading
- a short intro if needed
- 1 to 3 paragraphs
- optional CTA or transition line

### Avoid:
- giant walls of uninterrupted text
- too many tiny fragmented paragraphs
- forcing every section into cards

Text sections should feel like readable thought, not content blocks assembled mechanically.

---

## GRID SECTION RULES

Use grids for:
- 12 pillars
- features
- three-part explanation blocks
- system categories
- future pathways

### Rule 1
Keep grid cards structurally consistent.

### Rule 2
Do not overload cards with too much text.

### Rule 3
Use grids to support scanning, not replace depth.

### Rule 4
Allow strong spacing between cards.

---

## TWO-COLUMN SECTION RULES

Use two-column layouts when showing:
- philosophy and implementation
- doctrine and application
- system and example
- principle and product

### Rule 1
Keep the balance clear.
One column should not overwhelm the other.

### Rule 2
Stack cleanly on mobile.

### Rule 3
Use this layout to create meaningful contrast, not visual busyness.

---

## TIMELINE / PATHWAY SECTION RULES

Use timeline blocks for:
- transition path
- roadmap
- phased development
- build sequences

### Rule 1
Keep each step short and clear.

### Rule 2
Do not over-design the timeline.

### Rule 3
The structure should help understanding, not distract from it.

---

## CTA SECTION RULES

Every major page should end with a clear next step.

### CTA sections should include
- a strong closing line
- a primary next action
- optional secondary path

### CTA tone
- inviting
- clear
- calm
- aligned

### Avoid
- urgency tactics
- pressure language
- generic marketing CTA language

---

## VISUAL BALANCE RULES

### 1. Calm over clutter
Do not crowd pages with too many visual treatments.

### 2. Structure over novelty
Consistency matters more than clever one-off layouts.

### 3. Light and dark balance
Use contrast gently.
Do not create harsh visual jumps between sections.

### 4. Content leads
Layout exists to support message, not overpower it.

### 5. MicroGrow integration
MicroGrow sections should feel like a natural part of the site, not like a different brand dropped into it.

---

## PAGE-SPECIFIC LAYOUT NOTES

### Home
The page should feel like an orientation flow:
vision -> problem -> pillars -> systems -> MicroGrow -> transition -> action

### Manifesto
The page should feel more reflective and flowing, but still structured enough to guide the reader.

### Blueprint
The page should feel architectural and organized.
This is one of the most structured pages in the site.

### Systems
The page should feel like a translation layer:
how the doctrine becomes living structures.

### MicroGrow
The page should feel grounded and practical, with stronger structural clarity and feature-oriented sections.

### About
The page should feel human and sincere, with slightly softer pacing.

---

## MOBILE LAYOUT RULES

Before finalizing any page, confirm:
- headings do not become oversized and awkward
- columns stack cleanly
- cards remain readable
- spacing still feels calm
- buttons are easy to tap
- no section feels compressed

A calm desktop design must remain calm on mobile.

---

## REUSABILITY RULE

When building a page section, ask:
- Will this pattern appear again?
- Can this become a reusable section?
- Can this be simplified?

If yes, add it to the reusable section library.

---

## FINAL PAGE TEST

Before approving a page layout, confirm:

1. Does the page feel guided rather than chaotic?
2. Is there enough breathing room?
3. Is the section order logical?
4. Does the page create a calm reading rhythm?
5. Does layout support meaning?
6. Would this still feel coherent if the site grows?

---

## FINAL RULE

A New Earth page should feel like entering a well-designed environment, not scrolling through noise.
